## How this file is organized

One section per **sub-topic** in this domain (use the sub-topic codes from
`topic-config.md` §3, e.g. `A.1`, `A.2`, …). Inside each sub-topic section,
group content by kind so it stays easy to scan:

- **Definitions** — short, precise, one item per term
- **Key Points** — numbered or bulleted lists of discrete facts
- **Formulas / Rules / Thresholds** — exact values, with units
- **Worked Examples (canonical)** — only short, reusable ones
- **Caveats / Common Confusions** — what trips learners up
- **Sources** — required for every concrete fact when verification level is
  `strict`; recommended otherwise

If a sub-topic has no content yet, leave its heading with a single line
`_No notes captured yet._` so the chapter outline stays complete.
